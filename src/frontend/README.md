# Сервис Frontend

Frontend — это веб-интерфейс для удобного просмотра манги. Он представляет собой пользовательский слой, построенный на основе FastAPI и шаблонов (например, Jinja2), и работает в связке с бэкенд-сервисом, предоставляющим данные о манге.

---

## 📌 Назначение

Сервис позволяет:
- Просматривать список доступной манги.
- Читать мангу через удобный веб-интерфейс.
- Взаимодействовать с бэкендом, который хранит и обрабатывает данные.

---

## ⚙️ Переменные окружения

Frontend поддерживает гибкую настройку через переменные окружения:

| Переменная         | Описание                                                                 | Значение по умолчанию       |
|--------------------|--------------------------------------------------------------------------|-----------------------------|
| `FRONTEND_HOST`    | Хост, на котором будет запущен frontend (для FastAPI).                   | `0.0.0.0`                   |
| `FRONTEND_PORT`    | Порт, на котором будет работать frontend.                                | `8000`                      |
| `API_URL`          | Полный URL бэкенда (например, `http://backend:8080`). Приоритет выше.   | —                           |
| `BACKEND_PORT`     | Порт бэкенда, если он запущен на том же хосте. Используется, если нет `API_URL`. | `8080`                |

> **Важно**:  
> - Если заданы обе переменные (`API_URL` и `BACKEND_PORT`), приоритет имеет `API_URL`.  
> - Если `API_URL` не указан, используется `http://localhost:<BACKEND_PORT>`.  
> - Если ни одна из переменных не задана, `BACKEND_PORT` автоматически устанавливается в `8080`.

---

## 🧩 Примеры использования

### 1. Бэкенд на том же сервере (стандартный случай)
```bash
FRONTEND_PORT=8000
BACKEND_PORT=8080
```
→ Frontend будет обращаться к http://localhost:8080 за данными.

### 2. Бэкенд на удалённом сервере
```bash
FRONTEND_PORT=8000
API_URL=http://api.manga-service.com:8080
```
→ Frontend будет использовать указанный внешний URL, игнорируя BACKEND_PORT

### 3. Полная кастомизация
```bash
FRONTEND_HOST=127.0.0.1
FRONTEND_PORT=3000
API_URL=https://api.example.com/v1
```
→ Frontend запустится на http://127.0.0.1:3000, а API-запросы будут идти на https://api.example.com/v1.
