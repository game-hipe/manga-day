# Модуль **bot** — что это и как он работает?

Данный модуль представляет собой двойного Telegram-бота:  
- **Административный бот** — для управления системой.  
- **Пользовательский бот** — для взаимодействия с конечными пользователями.

Этот README поможет быстро разобраться в настройке и особенностях работы модуля.

---

## 📌 Общие особенности

- **PROXY** — единый параметр прокси для обоих ботов.  
  Если указан, будет использоваться как административным, так и пользовательским ботом.
  
- **Токены**:  
  Программа может запуститься, если указан хотя бы один из токенов:
  - `$ADMIN_BOT_TOKEN` **или**
  - `$USER_BOT_TOKEN`  
  Если оба отсутствуют — приложение завершится с ошибкой.

---

## 🔐 Административная часть

Админ-бот предоставляет интерфейс для управления, просмотра логов, рассылок и других задач.

### Обязательные переменные окружения:

```bash
ADMIN_BOT_TOKEN=ABCD:1234         # Токен Telegram-бота (BotFather)
ADMIN_USERNAME=admin              # Логин для входа в админку
ADMIN_PASSWORD=admin              # Пароль для входа в админку
ADMIN_IDS=12345,12346             # Список chat_id администраторов (через запятую)
```
> ⚠️ Убедитесь, что указанные пользователи действительно являются администраторами. > Только они смогут получать оповещения и использовать команды.


## 🧑 Пользовательская часть
Пользовательский бот служит точкой входа для клиентов. Он взаимодействует с API и генерирует PDF-документы.

Обязательные переменные окружения:
```Bash
USER_BOT_TOKEN=ABCD:1234          # Токен Telegram-бота для пользователей
API_URL=http://127.0.0.1:8080     # Бэкенд API (обработка данных)
PDF_URL=http://127.0.0.1:3040     # Сервис генерации PDF (см. microservice/pdf)
```
Опциональные переменные:
```Bash
SITE_URL=http://127.0.0.1:8000    # Ссылка на сайт (например, в кнопках)
```

> ℹ️ Подробнее о сервисе генерации PDF см. в документации модуля microservice/pdf.


## 🛠 Пример .env файла
```ENV
# Прокси (опционально)
PROXY=socks5://user:pass@host:port

# Административный бот
ADMIN_BOT_TOKEN=your_admin_token_here
ADMIN_USERNAME=admin
ADMIN_PASSWORD=secure_password
ADMIN_IDS=123456789,987654321

# Пользовательский бот
USER_BOT_TOKEN=your_user_token_here
API_URL=http://localhost:8080
PDF_URL=http://localhost:3040
SITE_URL=http://localhost:8000
```
## 📎 Замечания
Все URL должны быть доступны из контекста бота (проверьте сетевые настройки).
Используйте .env файл для хранения конфигурации — не коммитьте его в репозиторий!
Для безопасности измените стандартные ADMIN_USERNAME и ADMIN_PASSWORD.
